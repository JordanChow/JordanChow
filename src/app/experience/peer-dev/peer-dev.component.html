Absence Calendar View

My next task was implementing a major feature - an absence calendar view. This would be a new component in the website that allows all employees to view their absences and statutory holidays in a calendar view for any selected year. For managers, they are able to select a team and view all the absences for the employees on that team.


This feature was crucial to optimize the efficiency of the absence viewing process. With this feature, employees can easily see their requests as well as stat holidays. For managers, they can quickly see who on their team will be absent when which will help for project management. The new feature will be implemented into production, during the next release, impacting all of PEER Group.

This component was developed in Angular and leveraged the Clarity UI Design System to help bootstrap the screen. To accomplish this, Angular resolvers and services were used to inject data into routes and receive data from the backend to be displayed in a user friendly manner. The calendar supports customization for the users in the legend. Any color can be chosen and the default colors can be restored with a reset button.

Automated File Transfer Utility

The Problem

The next problem I was faced with was finding a way to upload and download files from an external ticketing site (Autodesk PLM). For each ticket, there can be attachments associated with ticket. Due to domain restrictions, these attachments can only be downloaded or uploaded from a virtual machine. My job was to find a way to automate the process of downloading relevant attachments (based on a customizable query) and upload them to a file share site that can easily be access from PEER's home computers. The opposite was also required - PEER employees can upload to the file share site and those files will then get uploaded to the correct ticket in the PLM site.


The Solution

To solve this problem, I decided to use Selenium WebDriver to simulate a user to access the PLM site, CrushFTP API to make HTTP requests for download and upload, and LiteDB to store the file information. The solution is a console application which is run as a script. The script checks the site for new files based on the database and upload or download the necessary files. It was set up to be executed on a scheduled basis on the VM using Window's Task Scheduler.
